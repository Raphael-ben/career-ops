# Mode: rb/write-cv — CV Writer

## Purpose
Generate a filled LaTeX CV tailored to a specific job description. Reads `profile_bank.json` and `cv-template-rb-lm.tex`, produces a filled `Raphael-Benyamine_CV.tex` in the output folder. Every slot in the template is filled — no `%%FILL:` markers remain.

## Inputs required before starting
- Job description text (provided by pipeline or pasted by user)
- Classification output: `role_type`, `industry`, `seniority`, `language`, `swiss` (true/false), `chopard_eligible`
- `profile_bank.json` path: `~/Claude-code/job-hunter-data/config/profile_bank.json`
- CV template: `~/Claude-code/job-hunter-data/templates/cv-template-rb-lm.tex`
- Template instruction file: `~/Claude-code/job-hunter-data/templates/cv-template-rb-lm.md`
- Output folder path (created by pipeline)

## Process

### Step 1 — Load inputs

Read `{output_folder}/profile-filtered.json` (pre-filtered profile for this job, generated by pipeline Step 3.5). Read `~/Claude-code/job-hunter-data/templates/cv-fill-spec.md` for LaTeX syntax reference. Read `~/Claude-code/job-hunter-data/templates/cv-template-rb-lm.tex` to copy the preamble verbatim when writing output. **Do not read `cv-template-rb-lm.md`** — all fill conventions are already in this mode file.

### Step 2 — Extract JD keywords (ATS injection)

Extract 15-20 keywords from the JD: noun phrases and verb phrases the JD uses to describe required skills and experience. These will be woven naturally into experience bullets. Do NOT invent experience — only reformulate existing profile_bank content using JD vocabulary.

### Step 3 — Select and order experience blocks

**Mandatory blocks — assign bullets first:**

| ID | Include | Bullets |
|---|---|---|
| `exp_pwc` | Always | 4–5 bullets |
| `exp_serpentine` | Always | 3 for finance/VC/strategy; 2 for all others |
| `exp_rolex_2` | Always | 3 bullets; trim to 2 for pure finance |
| `exp_rolex_1` | Always | 2 bullets for technical/industrial roles; 1 bullet for all others |

**Space-fill rule — MANDATORY, run after assigning mandatory bullets:**

Step A — compute base count:
```
base_count = PwC_bullets + Serpentine_bullets + Rolex2_bullets + Rolex1_bullets + 1 (MBA) + 1 (IFMA)
```

Step B — decide Chopard. Two independent conditions, EITHER triggers inclusion:
1. `chopard_eligible: true` in classification
2. `base_count < 15`

If EITHER is true → include `exp_chopard` with 2 bullets. **`chopard_eligible: false` does NOT override condition 2.** Check both explicitly.

Step C — decide Prépa. Two independent conditions, EITHER triggers inclusion:
1. Role is France-facing (language = fr, or company is French)
2. `(base_count + Chopard_bullets_if_added) < 15`

If EITHER is true → include `edu_prepa`.

This prevents a half-empty page: whenever base_count < 15, Chopard activates automatically regardless of sector eligibility. They are suppressed only when the mandatory blocks already produce ≥ 15 bullets and the role is not luxury/France-facing.

**Education selection:**
- `edu_hsg` (MBA): Always — 1 bullet (CGPA + ranking line)
- `edu_ifma` (IFMA engineering): Always — 1 bullet
- `edu_prepa` (Lycée La Martinière, Classes Préparatoires): Include if the space-fill rule above triggers it, or if the role is France-facing

Achievement selection: within each role, only surface achievements where `surface_in` includes at least one of the role's classifications. Prefer `confidence: "high"` achievements; use `"medium"` sparingly.

Title for `exp_serpentine` MUST be "Venture Scout / Analyst" — never "Analyst" or "Venture Analyst".

### Step 4 — Apply standing rules

Read `profile_bank.rules`. Apply all constraints:

- **never_mention**: Never use any term in this list. Currently: "VBA". Apply everywhere — bullets, sidebar, anywhere. Even if a template instruction example shows "VBA", do not put it in the actual content.
- **Framing rules**:
  - `exp_rolex_1` is NEVER described or framed as an internship. Use the actual role title.
  - `exp_serpentine` title exactly "Venture Scout / Analyst" — preserved verbatim.
- **Tense**: `exp_pwc` uses present tense (`end_status: "contract_ending"` → still employed). Never write "left PwC", "former PwC", or any past framing for PwC. All other roles use past tense.
- **Conditional mentions**: `exp_chopard` only if `chopard_eligible: true`. Aviation/PPL only for aviation/finance/industrial/manufacturing classifications.

### Step 5 — Compose sidebar content

**FOREIGN LANGUAGES** (from `profile_bank.candidate.languages`):
Map `level` to English: `native` → "Mother tongue", `C2` → "Bilingual", `C1` → "Fluent", `B2` → "Professional". Use `\sline` (all items are too long for `\pair`).

**PERSONAL SKILLS** (6-10 items from `skills_master.soft` + `skills_master.domain` qualitative items):
Short 1-3 word items. Mix both groups. Tailor to classification: industrial/luxury → emphasize Manufacturing, Operations, Process; consulting/strategy → Business Analysis, Due Diligence, Communication. Use `\pair` for short items (≤18 chars), `\sline` for longer ones.

**COMPUTER SKILLS** (5-9 items from `skills_master.tools` + technical end of `skills_master.technical`):
Tool and language names only. NEVER include a tool not in `profile_bank.skills_master.tools`. Never mention any term from `never_mention`. Use `\pair` and `\sline` per length rules.

**EXTRA-CURRICULAR ACTIVITIES** — always include this block. Fixed content:
- CrossFit Level 1 trainer — always include
- Aviation -- PPL(A) — include only if classification.industry is one of: aviation, finance, industrial, manufacturing, energy, consulting
- Skiing / Mountaineering — include if the above gives ≥ 2 items, to fill the block
- Triathlon / Cycling — include alongside Skiing if space allows
- Rock Climbing / Golf — include alongside Triathlon if space allows

Minimum block: at least CrossFit + one more item. Never emit an empty block.

Sidebar `\pair` / `\sline` decision: items ≤ ~18 characters → `\pair`; longer → `\sline`. One leftover short item at end of a group → `\sline`.

### Step 6 — Fill the template

Filling conventions from `cv-template-rb-lm.md`:

**User variables block** — fill at top of preamble:
```latex
\newcommand{\CVName}{Raphael Benyamine}
\newcommand{\CVTagline}{Line 1 positioning,\\[2pt]Line 2 if needed}
\newcommand{\CVPhoto}{cv-2.jpg}
\newcommand{\CVAddress}{Schaffhauserstrasse 75\\8057 Zürich}
\newcommand{\CVPhone}{+41\,78\,621\,34\,68}
\newcommand{\CVEmail}{raphael.benyamine@gmail.com}
```

**Tagline**: 1-2 lines positioning Raphael for this specific role. Use JD's language register. **Max ~22 characters per line** — the sidebar renders the tagline at `\large\textbf` in a 58 mm column, which holds ≈ 22 characters before wrapping. Count characters before writing. If the title line would exceed 22 chars, shorten or abbreviate (e.g. "Program Manager" not "Program Manager, Professional Services").

**Experience blocks** — one per selected role, most recent first:
```latex
\begin{experience}{date range}{job title}{company}{location}{company URL}
  \item Bullet text
  \item Bullet text
\end{experience}
```

Date format: `Month.~'YY -- Month.~'YY` for completed; `Month.~'YY -- present` for current. Abbreviated months: Jan., Feb., Mar., Apr., May, June, July, Aug., Sept., Oct., Nov., Dec.

Special chars in bullet text: `\&` for ampersand, `\%` for percent, `\_` for underscore. No curly quotes — use straight ASCII. No `\vspace` anywhere in the body.

**Education blocks** — after `\vspace{\sectionGap}` + `\mainSection[\faGraduationCap]{EDUCATION}`:
```latex
\begin{education}{date range}{degree}{institution}{location}{URL}
  \item Note or distinction
\end{education}
```

### Step 7 — ATS keyword injection

Integrate the 15-20 JD keywords naturally into filled bullets. Reformulate existing achievements using JD vocabulary. Never invent experience that isn't in profile_bank.

### Step 8 — Swiss German orthography

If `swiss: true`: use `ss` instead of `ß` throughout all bullet text, sidebar items, and tagline. generate-latex-rb.mjs will apply a second pass, but the mode output must already be clean.

If `swiss: false` and company is German (DE): use standard `ß`.

### Step 9 — Write output

Write the filled `Raphael-Benyamine_CV.tex` to `{output_folder}/Raphael-Benyamine_CV.tex`.

Report:
- EXP blocks included (list of `exp_id`s)
- Keyword list injected
- Chopard included: yes/no and why
- Any standing rules triggered

## Hard rules — violations cause verifier to fail

1. **Source-bound claims.** Every bullet maps to a specific achievement in `profile_bank`. No claim that cannot be traced to an `exp_id` or `ach_id`.
2. **No invented metrics.** If profile_bank says "led workstream", do not write "led 5-person workstream". Numbers only if in profile_bank.
3. **No inferred sector specifics.** Do not claim "engagements in [sector from JD]" unless profile_bank explicitly states it.
4. **No banned terms.** Never "VBA" anywhere.
5. **Tense compliance.** PwC = present. Everything else = past.
6. **Single page.** If content overflows, trim bullets — do not adjust geometry.
7. **Never devalue the profile.** No "without hierarchical power", no "the project wasn't technically complex", no "I did not have...".
